# 🌐 글로벌 자산배분 백테스터

글로벌 ETF 및 국내 주식을 대상으로 **분배금 재투자 + 매월 추가 불입**을 포함한 포트폴리오 백테스팅을 원화 기준으로 시뮬레이션하는 Streamlit 웹 앱입니다.

---

## 📌 주요 기능

- **멀티 자산 포트폴리오**: 미국 ETF와 국내 주식(KOSPI/KOSDAQ)을 자유롭게 혼합
- **분배금(배당금) 자동 재투자**: 수령한 분배금을 해당 종목에 즉시 재투자하여 복리 효과 반영
- **매월 추가 불입 시뮬레이션**: 정기 적립식 투자 전략 검증
- **원화 환산**: 미국 자산은 USD/KRW 환율을 자동 적용해 원화로 통합 계산
- **핵심 성과 지표**: 최종 평가금액, 수익률, CAGR, MDD, 누적 분배금
- **분배금 상세 리포트**: 종목별 × 월별 분배금 테이블 및 막대 그래프
- **시작일 자동 조정**: 가장 늦게 상장된 종목 기준으로 공통 시작일 자동 설정

## 🖥️ 사용 방법

### 사이드바 입력 항목

| 항목 | 설명 | 예시 |
|---|---|---|
| **투자 종목** | 쉼표로 구분된 티커 목록 | `SPY, TLT, GLD, DBC` |
| **배분 비중** | 합계 100%의 정수 비중 | `25, 25, 25, 25` |
| **초기 투자 금액** | 최초 투자 원금 (원화) | `10,000,000` |
| **매월 추가 불입금** | 월 정기 적립금 (0이면 거치식) | `300,000` |
| **시작일 / 종료일** | 백테스트 기간 설정 | `2020-01-01 ~ 오늘` |

### 티커 입력 규칙

- **미국 ETF/주식**: 티커 그대로 입력 (`SPY`, `QQQ`, `AAPL`)
- **KOSPI 종목**: 종목코드 6자리 입력 시 `.KS` 자동 추가 (`005930` → `005930.KS`)
- **KOSDAQ 종목**: `.KQ`를 직접 붙여서 입력 (`086520.KQ`)

## 📊 출력 결과

### 성과 지표 (상단 6개 카드)

| 지표 | 설명 |
|---|---|
| **최종 평가 금액** | 백테스트 종료 시점 포트폴리오 평가액 |
| **수익률** | (최종금액 / 총투입원금 - 1) × 100 |
| **총 투입 원금** | 초기금액 + (월불입금 × 경과 월수) |
| **CAGR** | 연평균 복합 수익률 |
| **MDD** | 고점 대비 최대 낙폭 |
| **누적 분배금** | 전 기간 수령한 분배금 합계 (원화) |

### 차트

- 📈 **자산 성장 곡선**: 일별 포트폴리오 가치 추이 (원화)
- 📊 **월별 분배금 막대 그래프**: 종목별 월간 분배금 수령 현황
- 💱 **환율 추이**: USD/KRW 환율 변화

## 🐛 버그 수정 이력 (v0 → v0-fixed)

원본 `etf_div_r0.py`에서 발견된 문제점과 수정 내용입니다.

### BUG-1 · 티커/비중 수 불일치 검증 없음
- **문제**: 종목 4개, 비중 3개 입력 시 `numpy` 브로드캐스팅 오류로 앱 크래시
- **수정**: 파싱 직후 `len(tickers) != len(weights)` 체크 및 `st.stop()` 추가

### BUG-2 · 비중 합계 100% 검증 없음
- **문제**: 합계가 100%가 아닌 비중 입력 시 결과가 무음으로 왜곡됨
- **수정**: `abs(sum(weights_raw) - 100.0) > 0.1` 조건으로 오류 안내 후 중단

### BUG-3 · 비중 파싱 예외처리 없음
- **문제**: 문자 입력 시 `ValueError`가 사용자에게 raw traceback으로 노출
- **수정**: `try/except ValueError`로 감싸 친화적 에러 메시지 출력

### BUG-4 · `@st.cache_data`에 `date` 객체 직접 전달
- **문제**: `datetime.date` 객체는 캐시 해시가 불안정하여 불필요한 재실행 유발
- **수정**: 함수 내부 첫 줄에서 `str(start)`, `str(end)`으로 변환

### BUG-5 · `tz_localize(None)` 오용
- **문제**: timezone-aware 인덱스에 `tz_localize(None)` 호출 시 `TypeError` 발생  
  (`localize`는 tz-naive에만 사용 가능)
- **수정**: `tz_convert(None)`으로 교체 (aware → naive 변환에 올바른 메서드)

### BUG-6 · `num_months` 계산 불안정
- **문제**: `resample('ME')`는 pandas 2.2 미만에서 `ValueError` 발생  
  (구버전은 `'M'` 사용, 2.2+는 `'ME'`로 변경됨)
- **수정**: `resample` 제거 후 `(last.year - first.year)*12 + (last.month - first.month)` 직접 계산으로 버전 무관하게 안정 동작
